范冰的二次学习

花里胡哨的各种 AI 产品满天飞，但我最近回归了命令行界面（CLI，Command Line Interface），转向 Claude Code / Codebuddy 这类新时代的编程工具。  
  
大概也是因为我到了容易怀旧的中登年纪，看到小时候自己在学习机 / DOS / QBasic 界面里玩的那一套界面，又文艺复兴了，有点熟悉有点兴奋。  
  
黑漆漆的命令行里，字符串的跳跃、闪动，很有《黑客帝国》的既复古又未来的科技感（BTW，「字节跳动」这名字怎么来的，想必老程序员张一鸣也是很有共鸣）。  
  
这一次从 Cursor / Windsurf / Google Antigravity 切回用 Claude Code / Codebuddy，不光只是酷炫赶时髦。  
  
起初契机，是我意识到，现在 AI 写代码已经太强大、太迅捷了。唯一拖累它的，是我操作的速度——我得在多窗口间切换，我得精准地用鼠标指针戳到页面上某个细小的按钮，我得克服种种推荐算法的诱惑。  
  
到头来，忘了自己一开始要干嘛，原本是在做什么任务。我才是那个拖累 AI 写代码的元凶。  
  
那些花里胡哨的 UI 设计，本质上是注意力收割机。每个按钮、每个颜色、每个动画、每个推荐，都在争夺人的认知资源。  
  
但是 ——  
  
\- 在命令行的终端里，我可以更聚焦地输入、等待、接收。没有标签页切换的视觉噪音，没有突然弹出的通知干扰，更没有「你可能感兴趣」的推荐算法投喂；  
  
\- 让命令行里的 Claude Code / Codebuddy 自动去做任务，也能空出我的鼠标键盘。我可以用它们做别的事，而不是鼠标停了、任务就断了（自动化脚本 / RPA 工具，比如影刀，也有这个问题，会占用鼠标键盘）；  
  
\- 最近 Claude Code / Codebuddy / Trae 相继推出的 Skill，也很强力。以前 RAG、MCP（包括需要拖拽节点的工具，比如 n8n）这些，我都只是看看，笑而不语，根本没花力气折腾。因为一早已经做出了判断：都是 AI 大模型本身能力还不够强阶段的中间产物，是混沌的、不稳定的、前期投入 ROI 很低的、需要生态来共创且短期难见回报的。迟早被模型进化迭代掉。今天看果然如此。但 Skill 确实扎实、强悍、实用、可复用。外挂在 CLI 里，直接将操作效率倍增。我直接吹爆。  
  
感兴趣的朋友试一把 CLI 吧，应该不会让你失望滴。
